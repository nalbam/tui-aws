# Refactor Skill

## Principles
- Improve structure without changing behavior
- Single Responsibility Principle (SRP)
- Remove duplicate code (DRY)
- Small, incremental steps with verification

## Process
1. **Analysis** — Identify target code, tests, callers, dependencies
2. **Plan** — Present what will change, what won't, risk assessment
3. **Execute** — Small verifiable steps, run tests after each
4. **Verify** — All tests pass, no behavior changes
